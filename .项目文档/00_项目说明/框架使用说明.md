# 框架使用说明

## 项目概述

小红书竞品笔记监控系统是一个基于 Next.js (App Router) + antd + Supabase 的全栈Web应用。

## 项目业务说明

### 核心功能
- 从千瓜数据平台获取按品牌归类的笔记列表
- 获取笔记前端数据、达人信息、合作费用
- 通过AI对内容分析，进行内容类别判别和笔记相关单品名称识别
- 自定义报告所需组织笔记，给出分析结果

### 数据流程
- 数据由独立的爬虫系统直接写入Supabase数据库表
- 本系统负责读取和分析这些数据
- 用户通过Web界面进行数据查看、分析和报告生成

## 技术架构

### 前端
- **框架**: Next.js 14+ (App Router)
- **UI组件库**: antd 5.x
- **语言**: TypeScript
- **样式**: CSS Modules + Tailwind CSS (可选)

### 后端
- **数据库**: Supabase PostgreSQL
- **认证**: Supabase Auth
- **API**: Supabase REST API + Edge Functions
- **存储**: Supabase Storage

### AI服务
- **AI引擎**: Google Gemini API
- **配置**: 通过环境变量配置API Key

### 部署
- **前端部署**: Vercel
- **后端服务**: Supabase Cloud

## 开发环境设置

### 1. 安装依赖
```bash
npm install
```

### 2. 配置环境变量
创建 `.env.local` 文件，参考 `.env.local.example`：
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
GEMINI_API_KEY=your_gemini_api_key
```

### 3. 启动开发服务器
```bash
npm run dev
```

访问 http://localhost:3000

### 4. 构建生产版本
```bash
npm run build
npm start
```

## 项目结构

```
/
├── app/                    # Next.js App Router 页面
│   ├── (auth)/            # 认证相关路由组
│   ├── (dashboard)/       # 主应用路由组
│   └── layout.tsx         # 根布局
├── components/            # 可复用组件
│   └── layout/            # 布局相关组件
├── lib/                   # 工具函数和配置
│   └── supabase/          # Supabase客户端
├── middleware.ts          # Next.js中间件（路由保护）
└── .项目文档/             # 项目文档目录
```

## 开发规范

- 使用TypeScript进行类型安全开发
- 遵循ESLint和Prettier代码规范
- 组件使用函数式组件和Hooks
- 页面路由使用Next.js App Router
- UI组件优先使用antd组件库
