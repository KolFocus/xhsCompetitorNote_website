# 测试文档

## 目录说明

本目录用于存放项目的测试相关文档，包括：
- 测试计划
- 测试用例
- 测试报告
- 数据库测试脚本
- API测试文档
- 测试数据（包含Note List测试数据）

## 文档结构

```
04_测试文档/
├── README.md                    # 本文件
├── 01_测试计划.md               # 测试计划文档
├── 02_测试用例.md               # 测试用例文档
├── 03_数据库测试.md             # 数据库相关测试
├── 04_API测试.md                # API接口测试
├── 05_测试报告/                 # 测试报告目录
└── 06_测试数据.md               # 测试数据（包含Note List测试数据）
```

## 测试类型

### 1. 单元测试
- 组件测试
- 工具函数测试

### 2. 集成测试
- API接口测试
- 数据库操作测试

### 3. 端到端测试
- 用户流程测试
- 功能完整性测试

### 4. 数据库测试
- 表结构验证
- 约束检查
- RLS策略测试

## 测试环境

- 开发环境：本地开发服务器
- 测试环境：Supabase测试数据库
- 生产环境：Supabase生产数据库

---

## 与“04_质量保证”的边界

- 本目录（04_测试文档）侧重“测试过程产物与执行”：计划、用例、执行记录、报告、测试数据与脚本。
- `04_质量保证` 目录侧重“质量体系与策略”：质量标准、评审规范、准入准出标准、度量指标、抽检策略、复盘模板等。
- 二者关系：质量保证定义标准与过程要求；测试文档承接执行并产出过程与结果。
