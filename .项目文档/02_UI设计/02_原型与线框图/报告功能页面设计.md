# 报告功能页面设计

## 1. 设计概述

本文档详细描述创建报告功能的UI原型设计，包括创建报告对话框、追加笔记对话框、报告详情页和分析报告页的具体设计方案。

### 1.1 设计原则
- 遵循 antd 5.x 设计规范
- 保持与现有系统风格一致
- 简洁明了的交互流程
- 清晰的视觉反馈

### 1.2 技术组件
使用 antd 5.x 组件库，包括 Modal、Form、Select、DatePicker、Table、Tabs、Button 等组件。

## 2. 创建报告对话框

### 2.1 布局结构

```
┌─────────────────────────────────────────────────────┐
│  创建报告                                    [×]    │
├─────────────────────────────────────────────────────┤
│                                                      │
│  报告名称 *                                          │
│  ┌────────────────────────────────────────────┐   │
│  │ 请输入报告名称（8-20字）                    │   │
│  └────────────────────────────────────────────┘   │
│                                                      │
│  品牌选择 *                                          │
│  ┌────────────────────────────────────────────┐   │
│  │ 请选择品牌（至少选择1个）          [▼]      │   │
│  └────────────────────────────────────────────┘   │
│                                                      │
│  时间范围                                            │
│  ┌──────────────┐  至  ┌──────────────┐          │
│  │ 开始日期     │      │ 结束日期     │          │
│  └──────────────┘      └──────────────┘          │
│                                                      │
│  ──────────────────────────────────────────────── │
│                                                      │
│  共找到 0 条笔记                                      │
│                                                      │
├─────────────────────────────────────────────────────┤
│                                    [取消] [创建报告] │
└─────────────────────────────────────────────────────┘
```

### 2.2 详细规格

#### 2.2.1 对话框规格
- **宽度**: 600px（参考设计规范中的中对话框规格）
- **位置**: 居中显示

#### 2.2.2 表单字段规格

**报告名称字段**
- **标签**: "报告名称" + 红色星号（必填标识）
- **输入框**: 
  - 占位符："请输入报告名称（8-20字）"
  - 验证规则：8-20个字符，实时校验
- **校验提示**: 
  - 未输入时："请输入报告名称"
  - 少于8字时："报告名称至少需要8个字符"
  - 超过20字时："报告名称不能超过20个字符"

**品牌选择字段**
- **标签**: "品牌选择" + 红色星号（必填标识）
- **下拉多选框**: 
  - 占位符："请选择品牌（至少选择1个）"
  - 支持搜索功能
  - 显示已选品牌数量（如：已选择 3 个品牌）
  - 支持全选/取消全选
- **校验提示**: 
  - 未选择时："请至少选择1个品牌"
  - 选择后：显示已选品牌列表（标签形式）

**时间范围字段**
- **标签**: "时间范围"（无星号，可选）
- **日期选择器**: DatePicker.RangePicker
  - 格式：YYYY-MM-DD（参考设计规范）
  - 支持快捷选择（今天、昨天、最近7天、最近30天等）
- **默认值**: 空（不选择）

#### 2.2.3 笔记数量显示

- **位置**: 表单字段下方，独立一行
- **样式**: 使用次要文字色（参考设计规范）
- **显示内容**: 
  - 计算中显示："正在计算..."
  - 计算完成显示："共找到 X 条笔记"（X 为数字）
  - 无笔记时显示："共找到 0 条笔记"（灰色提示）

#### 2.2.4 按钮规格

- **位置**: 对话框底部右侧
- **按钮样式**:
  - **取消按钮**: 次要按钮样式（参考设计规范）
  - **创建报告按钮**: 主要按钮样式（参考设计规范）
- **按钮状态**:
  - **禁用状态**: 
    - 报告名称未填写或不符合要求
    - 未选择品牌
    - 正在计算笔记数量
    - 笔记数量为0
  - **可用状态**: 
    - 报告名称已填写（8-20字）
    - 至少选择1个品牌
    - 计算完成
    - 至少找到1条笔记
- **按钮文字**: 
  - 计算中："创建报告（计算中...）"
  - 正常："创建报告"

#### 2.2.5 交互行为

1. **品牌选择变化**: 
   - 自动触发笔记数量计算
   - 计算过程中按钮禁用
   - 显示"正在计算..."提示
   - 计算完成后显示结果并恢复按钮

2. **时间范围变化**: 
   - 自动触发笔记数量计算
   - 计算过程中按钮禁用
   - 计算完成后显示结果并恢复按钮

3. **报告名称输入**: 
   - 实时校验字符长度
   - 显示校验提示信息
   - 影响按钮可用状态

4. **点击创建报告**: 
   - 显示加载状态
   - 批量导入笔记
   - 成功后关闭对话框并跳转到报告详情页

- **位置**: 对话框底部右侧
- **按钮样式**:
  - **取消按钮**: 次要按钮样式（参考设计规范）
  - **确认追加按钮**: 主要按钮样式（参考设计规范）
- **按钮状态**:
  - **禁用状态**: 
    - 未选择品牌
    - 正在计算笔记数量
    - 将新增笔记数量为0
  - **可用状态**: 
    - 至少选择1个品牌
    - 计算完成
    - 将新增至少1条笔记
- **按钮文字**: 
  - 计算中："确认追加（计算中...）"
  - 正常："确认追加"

#### 3.2.5 交互行为

1. **品牌选择变化**: 
   - 自动触发增量笔记数量计算
   - 计算过程中按钮禁用
   - 显示"正在计算..."提示
   - 计算完成后显示结果并恢复按钮

2. **时间范围变化**: 
   - 自动触发增量笔记数量计算
   - 计算过程中按钮禁用
   - 计算完成后显示结果并恢复按钮

3. **点击确认追加**: 
   - 显示加载状态
   - 批量追加笔记到报告（自动去重）
   - 成功后关闭对话框并刷新报告详情页

**位置**: 统计信息卡片区域下方，筛选器区域上方

**布局**
- **样式**: 参考设计规范的卡片规范

- **操作按钮**: 右上角显示"导出Excel"、"配置"按钮（均为次要按钮样式；导出包含“汇总+明细”两个sheet）

**功能说明**
- 展示各层级达人的统计数据（达人数量、占比、平均粉丝数、总互动量（不含分享）、笔记数量、分享量/分享占比等）
- 支持配置自定义层级
- 详细设计参考：`02_UI设计/02_原型与线框图/达人矩阵分析页面设计.md`

**配置存储与读取（技术说明）**
- 配置存储位置：`qiangua_report.CustomLevels`（JSON）。
- 读取优先级：若存在 `CustomLevels` 则按其分层统计，否则使用默认分层。
- 相关接口：
  - 获取/保存配置：`app/api/reports/[id]/blogger-matrix/config/route.ts`
  - 统计数据：`app/api/reports/[id]/blogger-matrix/stats/route.ts`

## 6. 交互状态说明

### 6.1 加载状态

- **计算笔记数量**: 
  - 显示"正在计算..."文字
  - 按钮显示"（计算中...）"
  - 按钮禁用
- **创建/追加报告**: 
  - 按钮显示 loading 图标
  - 按钮文字保持不变
  - 按钮禁用
- **表格加载**: 表格显示 loading 状态，显示加载动画

### 6.2 错误状态

- **表单校验错误**: 输入框下方显示红色错误提示
- **API 错误**: 使用 message.error() 显示错误提示，提示信息清晰明确
- **网络错误**: 显示"网络错误，请稍后重试"提示，提供重试按钮

### 6.3 成功状态

- **创建报告成功**: 使用 message.success() 显示成功提示，自动跳转到报告详情页
- **追加笔记成功**: 使用 message.success() 显示成功提示，自动刷新报告详情页
- **操作成功**: 使用 message.success() 显示操作成功提示，刷新相关数据
