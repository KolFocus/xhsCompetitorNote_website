## 字段映射说明

### 1. 数据来源说明

- **getNoteList**: 从 `Data.ItemList[]` 数组中获取每个笔记对象的数据
- **getSimpleNote**: 从 `Data` 对象中直接获取数据
- **CooperateBindList**: 从 `ItemList[].CooperateBindList[]` JSON数组中提取品牌信息
- **博主信息**: 从 `ItemList[]` 中的博主相关字段提取，同时：
  - 同步到 `qiangua_blogger` 表（存储博主最新信息）
  - 冗余存储到 `qiangua_note_info` 表（以 `Blogger` 开头的字段，保留笔记发布时的快照数据）

### 2. 字段提取规则

#### BrandId、BrandIdKey、BrandName 提取规则
- 从 `CooperateBindList` JSONB 数组的第一个元素（索引0）中提取
- 如果 `CooperateBindList` 为空数组或不存在，这三个字段为 NULL
- 用于快速查询和关联，避免每次都需要解析 JSONB 字段

#### SmallAvatar 字段映射
- **qiangua_blogger 表**:
  - **getNoteList 接口**: 使用 `SmallAvatar` 字段
  - **getSimpleNote 接口**: 使用 `BloggerSmallAvatar` 字段
  - 两个字段本质相同，但字段名不同，需要统一映射到 `qiangua_blogger` 表的 `SmallAvatar` 字段
- **qiangua_note_info 表（冗余字段）**:
  - **getNoteList 接口**: 使用 `SmallAvatar` 字段映射到 `SmallAvatar`
  - **getSimpleNote 接口**: 使用 `BloggerSmallAvatar` 字段映射到 `SmallAvatar`
  - 作为冗余字段，保留笔记发布时的快照数据

### 3. 数据类型转换

| API字段类型 | API字段示例 | 数据库类型 | 转换说明 |
|------------|------------|-----------|----------|
| STRING (ISO 8601) | "2025-10-29T17:47:41+08:00" | TIMESTAMPTZ | 需要解析ISO 8601格式字符串（包含时区信息） |
| STRING (DATE) | "2025-10-29" | DATE | 需要解析YYYY-MM-DD格式字符串 |
| ARRAY | [{...}] | JSONB | 需要序列化为JSONB格式 |
| BIGINT | 1727038833 | VARCHAR(64) | NoteId、BloggerId：需要将数字转换为字符串 |
| INTEGER | 20251029 | VARCHAR(64) | DateCode：需要将数字转换为字符串（格式：YYYYMMDD） |
| INTEGER | 46899 | VARCHAR(64) | BrandId：需要将数字转换为字符串 |
| INTEGER | 3537 | VARCHAR(64) | McnInfoId：需要将数字转换为字符串 |

### 4. 空值处理

- 某些字段在API中可能为 `null` 或空字符串
- 数据库字段设置了默认值的，使用默认值
- 没有默认值的字段，允许为 NULL

### 5. 字段缺失处理

- **Content** 和 **XhsNoteUrl**: 仅通过 `getSimpleNote` 接口获取，首次同步时可能为空
- **CreatedAt** 和 **UpdatedAt**: 数据库自动管理，无需从API获取
- **BrandDescription** 和 **BrandLogo**: 暂无API提供，需要手动维护或等待接口扩展

## 数据同步流程映射

### 步骤1: 同步博主信息（getNoteList）
```
API: getNoteList → Data.ItemList[].博主相关字段
数据库: qiangua_blogger 表
操作: INSERT ... ON CONFLICT (BloggerId) DO UPDATE
说明: 需要先同步博主信息，保证外键约束的有效性
```

### 步骤2: 同步品牌信息（从CooperateBindList提取）
```
API: getNoteList → Data.ItemList[].CooperateBindList[]
数据库: qiangua_brand 表
操作: INSERT ... ON CONFLICT (BrandId) DO UPDATE
同时提取: BrandId, BrandIdKey, BrandName 到 qiangua_note_info 表
```

### 步骤3: 同步笔记列表（getNoteList）
```
API: getNoteList → Data.ItemList[]
数据库: qiangua_note_info 表
操作: INSERT ... ON CONFLICT (NoteId) DO UPDATE
关联字段: BloggerId (外键关联 qiangua_blogger 表)
冗余字段: 同时保存博主信息冗余字段（以Blogger开头的字段），保留笔记发布时的快照
注意: 冗余字段在首次插入时保存，后续更新时通常不更新，以保留历史快照
```

### 步骤4: 同步笔记详情（getSimpleNote）
```
API: getSimpleNote → Data
数据库: qiangua_note_info 表
操作: UPDATE ... WHERE NoteId = ?
更新字段: Content, XhsNoteUrl
可选冗余字段: BloggerNickName, SmallAvatar（如果接口返回了博主信息，注意：getSimpleNote 返回的是 BloggerSmallAvatar，需要映射到 SmallAvatar）
注意: getSimpleNote 接口返回的博主信息较少，如需完整快照信息，建议使用 getNoteList 接口
```

## 注意事项

1. **字段名大小写**: PostgreSQL 对大小写敏感，使用 PascalCase 字段名时，SQL语句中需要用双引号包裹
2. **ID字段类型转换**: 
   - `NoteId`、`BloggerId`：API返回BIGINT类型，需转换为VARCHAR(64)字符串存储
   - `BrandId`：API返回INTEGER类型，需转换为VARCHAR(64)字符串存储
   - `DateCode`：API返回INTEGER类型（如20251029），需转换为VARCHAR(64)字符串存储
   - `McnInfoId`：API返回INTEGER类型，需转换为VARCHAR(64)字符串存储
3. **时间格式**: `PublishTime` 和 `UpdateTime` 为 ISO 8601 格式字符串（包含时区信息，如：`2025-10-29T17:47:41+08:00`），需要转换为 TIMESTAMPTZ (TIMESTAMP WITH TIME ZONE) 类型
4. **日期格式**: `PubDate` 需要转换为 DATE 类型
5. **JSONB存储**: `CooperateBindList` 需要存储为 JSONB 格式
6. **价格单位**: `AdPrice` 单位为"分"，存储时注意单位
7. **图片URL**: 可能为相对路径（以 `//` 开头），需要补充协议（`https:`）
8. **冗余字段设计**: 
   - `qiangua_note_info` 表中的 `BloggerId`、`BloggerNickName`、`BloggerProp`、`BloggerTags`、`BloggerTagName` 以及 `Fans`、`LevelNumber`、`LevelName`、`Gender`、`Location`、`BigAvatar`、`SmallAvatar`、`McnName`、`McnInfoId`、`IsBrandPartner`、`OfficialVerified`、`GoodsCount`、`NoteActiveCount`、`AdPrice`、`AdPriceUpdateStatus`、`PriceType`、`LinkInfo` 等字段为冗余字段，用于存储博主在笔记发布时的快照数据
   - 这些字段的值在笔记首次同步时从API接口获取并保存，不会随 `qiangua_blogger` 表的更新而更新
   - 这样设计的好处是：方便查询（无需JOIN）、保留历史（粉丝数、价格等可能变化的数据）、性能优化（减少关联查询）
   - 在更新笔记时，冗余字段通常不更新，以保留历史快照。如需更新快照数据，可以显式地在更新时覆盖这些字段
9. **约束条件**: 
   - **NOT NULL 约束**:
     - `qiangua_note_info` 表：`NoteId`（主键）、`DateCode`、`PublishTime`、`BloggerId`（外键）
     - `qiangua_brand` 表：`BrandId`（主键）、`BrandIdKey`（唯一键）、`BrandName`
     - `qiangua_blogger` 表：`BloggerId`（主键）、`BloggerIdKey`（唯一键）、`BloggerNickName`
   - **外键约束**:
     - `qiangua_note_info.BloggerId` → `qiangua_blogger.BloggerId`（必填，NOT NULL）
     - `qiangua_note_info.BrandId` → `qiangua_brand.BrandId`（可选，可为NULL）
   - **可为空的字段**:
     - `qiangua_note_info` 表：`BrandId`、`BrandIdKey`、`BrandName` 均可为空（从 CooperateBindList 提取，如果不存在则为空）
   - **唯一约束**:
     - `qiangua_brand.BrandIdKey` 必须唯一
     - `qiangua_blogger.BloggerIdKey` 必须唯一
